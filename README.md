# DevOps Monitoring & Alerting System

## Overview
This project demonstrates a production-style monitoring and alerting system built using **Prometheus, Grafana, and a Python webhook service deployed on AWS EC2 with systemd**.

It simulates a real-world observability pipeline by monitoring CPU usage, triggering alerts when thresholds are exceeded, and logging incidents for auditing and response tracking.

---

## Architecture

### System Flow
CPU Stress Test → Node Exporter → Prometheus → Grafana → Webhook → Incident Log

![Architecture](screenshots/architecture.png)

---

## Monitoring Workflow

This system follows a full observability pipeline:

- CPU load is generated using `stress-ng`
- Node Exporter collects system-level metrics from the EC2 instance
- Prometheus scrapes and evaluates metrics against alert rules
- Grafana visualizes metrics and triggers alerts based on thresholds
- A Python webhook service receives alert notifications
- All alerts are logged into `incidents.log` for monitoring and auditing

---

## Key Features

- Real-time CPU monitoring using Node Exporter
- Grafana dashboards for system visualization
- Alerting rules for CPU threshold breaches
- Python-based webhook receiver for alert processing
- systemd service for production-grade reliability
- Persistent incident logging system
- Fully deployed on AWS EC2 (Ubuntu)
